## 一、为何现有 4 个算法不够

把现有 lgo_rrt.py lgo_hho.py lgo_dqn.py lgo_hybrid.py 通读下来，四个算法的核心流程完全相同：

`
对每架无人机：
  sa = service_areas[i % len(sa)]
  rp = rescue_points[i % len(rp)]
  safe_z = max(sa.z, rp.z) + margin
  path = [sa, mid_safe, rp]
  distance = sum(euclidean)
`

**本质**：只做飞越，没有真正的绕障路径规划。

飞越模式的问题：
- 低空救援（Z=9~73m）时，飞越高度可能穿过建筑
- 没有任务分配优化，多机可能挤向同一救援点
- 没有电量校验，飞行途中可能没电
- 没有多目标权衡，无法在最短路径和最安全路径之间取舍

文档里 14 个算法覆盖 6 大类能力，挑出 6 个补齐缺失的每一环。

### 5.1 algo_rrt_star.py — 3D 路径规划器

**职责**：给定起点和终点，在 3D 连续空间中搜索一条无碰路径。

**核心参数**：
- max_iter = 800
- step_size = 35
- goal_bias = 0.15
- rewire_radius = 60
- n_check = 12

**算法流程**：
`
1. 初始化: nodes=[start], parents=[-1], costs=[0]
2. 循环 max_iter=800 次:
   a. 以 goal_bias 概率采样目标点，否则在 bounds 内随机采样
   b. 找最近邻 nearest
   c. 以 step_size 为步长延伸 new_node
   d. 碰撞检测: 在 nearest->new_node 之间取 n_check=12 个采样点
      - 若任一采样点落在障碍物包围盒内(+5m margin) -> 丢弃
      - 山区场景额外检查: z < terrain_height(x,y) -> 丢弃
   e. ChooseParent: 在 rewire_radius 范围内找最优父节点
   f. Rewire: 反向检查新节点能否优化邻居的路径
   g. 若 new_node 距离 goal < step_size 且无碰 -> 结束，回溯路径
3. 返回路径或 None
`

**与现有 algo_rrt.py 的关键区别**：
- 新增 ChooseParent + Rewire: 这是 RRT* 渐近最优性的核心，现有版本只有 RRT 没有 *
- numpy 向量计算替代纯 math 运算，减少循环
- n_check=12 更密采样碰撞检测

**fallback 策略**：
若 RRT* 未收敛(返回 None)，改用飞越模式并标记 success_rate 降低，同时在 message 中提醒用户。

### 5.2 algo_risk3d.py — 风险评估器

**职责**：对 RRT* 给出的路径计算风险评分，高风险时在风险场中微调。

**三种风险**：
`
1. 静态建筑风险: 路径点到最近建筑中心的距离 d -> risk = exp(-d/tau)
2. 动态碰撞风险: 路径点到其他无人机航迹的最短距离 -> risk += exp(-d/tau2)
3. 地形碰撞风险: 路径点 z < terrain_height -> risk += 极大值(直接拒绝)
total_risk = sum(risk_i) / len(path)
`

**推荐实现方式**：作为 RRT* 的后处理模块嵌入 solve() 内部。即 solve() 先调 RRT* 得到路径，再计算风险，若 risk > threshold 则微调高风险路段。

### 5.3 algo_ialns.py — 任务分配器

**职责**：决定哪架无人机去哪个救援点，这是一个 VRP(车辆路径问题)。

**算法核心**：
`
1. 构建代价矩阵 cost[i][j] = distance(sa_i -> rp_j) / drone_speed + penalty(priority)
2. 初始解: pair assignment(每架无人机分配一个唯一救援点)
3. 自适应大邻域搜索(500~1000 次迭代):
   a. destroy(k=3): 随机移除 3 个分配
   b. repair: 对每个被移除点，找到代价增量最小(即 removed-inserted 差最大)的无人机和最优插入位置
   c. 若新解优于当前解 -> 接受; 否则按 Metropolis 准则概率接受
`

**代价函数(含优先级惩罚)**：
`python
penalty = {0: 0, 1: 50, 2: 200, 3: 1000}
cost = distance + penalty[priority]
`

**循环导入注意事项**：IALNS solve() 内部需要调 RRT* 为候选对算路径距离，应做延迟导入：
`python
def solve(self, drones, materials, service_areas, rescue_points, map_obj):
    from algo_rrt_star import Algorithm as RRTStarAlgo
    rrt = RRTStarAlgo()
    ...
`

**与文档版本的差异**：文档 IALNS 节点索引从 1 开始(node 0=depot)，本框架从 0 开始，移植时需适配。

### 5.4 algo_power.py — 电量约束器

**职责**：检查每段路径是否超出无人机电量航程，超出时插入换电站。

**算法核心**：
`
1. 参数: e_consume=0.08(每米), max_e=100(满电), swap_time=1.8(换电时间惩罚)
2. 对每条路径逐段累计:
   - 若当前电量不足以飞下一段:
     a. 将最近 service_area 作为换电站
     b. 在该点插入换电站航点(+1.8 时间惩罚)
     c. 重置电量=100%
3. 返回带换电站插入的最终路径
`

**调用方式**：是后置处理器，IALNS -> RRT* -> Power 顺序执行，不能独立使用。

### 5.5 algo_nsga2.py — 多目标选择器

**职责**：当最短路径和最安全路径冲突时，给出帕累托最优解集。

**三个优化目标**：
- f1 = 总飞行时间(秒)
- f2 = 总风险评分
- f3 = 总能耗(kWh)

**算法核心**：
`
1. 为每架无人机生成 3 条候选路径:
   - 候选 A: RRT* 最短路径(偏重 time)
   - 候选 B: Risk3D 最低风险路径(偏重 risk)
   - 候选 C: 折中路径(在 A/B 之间插值)
2. 对候选解做非支配排序 + 拥挤度计算
3. 从帕累托前沿选 knee point(拐点)作为推荐解
4. 返回推荐方案 + 2 个备选方案(在 message 中列出)
`

**工程简化**：不建种群遗传操作，而是对候选解做非支配排序，提供 3 个推荐方案。

### 5.6 algo_apf.py — 轨迹平滑器

**职责**：将离散路径点拟合成光滑可飞的曲线。

**算法核心**：
`
1. 引力场: F_att = -k_att * (pos - goal)
2. 斥力场: F_rep = k_rep * (1/d - 1/d0) * (1/d^2) * grad_d  当 d < d0
3. 步长迭代 30~50 步:
   pos_new = pos + lr * (F_att + F_rep) / |F_total|
4. 收敛后得到平滑曲线
`

**作为后处理器**：APF 不能独立运行，输入是 RRT* 输出的路径点，输出是平滑后的路径点。

## 六、关键工程决策

### 6.1 内部调用方式

每个算法是独立的 Algorithm 类，但实际 serve 的是一个完整 pipeline。

**推荐策略：各自独立**

每个 .py 都是独立可运行模块，内部自行调用其他模块的类。

`python
# algo_ialns.py 内部
class Algorithm(BaseAlgorithm):
    def solve(self, drones, materials, service_areas, rescue_points, map_obj):
        from algo_rrt_star import Algorithm as RRTStar
        from algo_risk3d import Algorithm as Risk3D
        rrt = RRTStar()
        risk = Risk3D()
        # 1. IALNS 分配
        # 2. RRT* + Risk3D 为每对规划
        # 3. Power 电量校验
        # 4. APF 轨迹平滑
        ...
`

### 6.2 全局 fallback 策略

RRT* 有概率 500 次迭代找不到解。必须有 fallback：

`python
path = self._rrt_star(start, goal, obstacles, bounds)
if path is None:
    safe_z = self._calc_cruise_z(start, goal, map_obj)
    path = [start, midpoint_at_safe_z, goal]
    success_rate *= 0.5
`

- fallback path 仍要标记 message: X 号无人机使用飞越模式(RRT* 未收敛)
- 在 3D 渲染中用红色/虚线区分 fallback path
