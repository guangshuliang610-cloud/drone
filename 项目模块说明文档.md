## 1. 项目总览

本系统是一个 PyQt5 桌面应用，用于模拟应急场景下无人机物资配送的调度决策。

**核心功能：**
- 两个场景：城市地震、山区避障
- 四种算法：HHO、RRT*、DQN、混合启发式
- 管理物资、服务区、救援点、无人机、任务
- 3D 地图可视化 + 路径规划演示

**技术栈：**
- Python 3.x + PyQt5（界面）
- Matplotlib（3D 地图渲染）
- NumPy（数学计算）
- JSON 文件存储（数据持久化）

### 5.2 `map_city_quake.py` — 城市地震场景

**作用：** 以天津为原型的北方城市震后场景。

**场景边界：** X[-420,420] Y[-390,390] Z[0,160]

**障碍物数据结构：**
```python
{
    "center": [x, y, z],    # 中心坐标
    "size": [w, h, d],      # 底面宽、底面高、建筑总高度
    "type": "tower"         # 类型：tower/highrise/midrise/building
}
```

**建筑类型：**
| 类型 | 数量 | 高度范围 | 说明 |
|------|------|---------|------|
| tower | 1 | 130m | 天津广播电视塔（天塔），城市地标 |
| highrise | 4 | 60-82m | CBD现代高层商务区 |
| midrise | 10 | 35-48m | 中层公共/商业建筑 |
| building | 15 | 13-20m | 传统低层居民区 |

**核心方法：**

```python
map = Map()

# 获取障碍物列表（算法避障用）
obstacles = map.get_obstacles()
# 返回：[{"center": [0,0,65], "size": [45,45,130], "type": "tower"}, ...]

# 获取服务区（无人机起飞点）
areas = map.get_service_areas()
# 返回：[{"name": "城北服务区", "x": 0.0, "y": 340.0, "z": 0.0, "scene": "城市地震场景"}, ...]

# 获取救援点（目的地）
points = map.get_rescue_points()
# 返回：[{"name": "居民区A-河北区", "x": -280.0, "y": 250.0, "z": 9.0, ...}, ...]

# 获取场景边界
bounds = map.get_bounds()
# 返回：((-420, 420), (-390, 390), (0, 160))

# 渲染3D地图
map.render_3d(ax)  # ax 是 matplotlib 的 Axes3D 对象
```

**渲染内容：**
- 地面平面（半透明深色）
- 网格道路系统（主干道 + 次干道）
- 30栋建筑（按类型不同颜色）
- 天塔特殊渲染（底座 + 塔身 + 观景台 + 天线）

### 5.3 `map_flood.py` — 山区避障场景

**作用：** 连续山脉地形 + 等高线，用于山地避障。

**场景边界：** X[-320,320] Y[-300,300] Z[0,220]

**障碍物：** 8座山体（高斯曲面），包围盒用于粗筛。

**核心方法：**

```python
map = Map()

# 获取山体包围盒（粗筛用）
obstacles = map.get_obstacles()
# 返回：[{"center": [-250,170,66], "size": [120,92,132], "type": "mountain"}, ...]

# 获取任意点的地形高程（精筛用，必须调用）
z = map.get_terrain_height(x_array, y_array)
# 输入：numpy 数组（标量或数组）
# 返回：对应的地形高度 z

# 获取等高线数据
contour_data = map.get_contour_lines(grid_size=170)
# 返回：{"levels": [...], "contours": [...], "grid": {"x": [...], "y": [...], "z": [[...]]}}
```

**山区避障必须两层判断：**
```python
# 第一层：包围盒粗筛
for obs in map.get_obstacles():
    cx, cy, cz = obs["center"]
    w, h, d = obs["size"]
    if (cx-w/2 <= px <= cx+w/2) and (cy-h/2 <= py <= cy+h/2) and pz <= d:
        return "碰撞"

# 第二层：地形高程精筛
terrain_z = map.get_terrain_height(px, py)
if pz < terrain_z:
    return "撞山"
```

### 6.5 `dispatch_page.py` — 救援调度（核心）

**作用：** 选择场景 + 算法，运行调度，显示 3D 路径结果。

**主要类：**

- `BaseAlgorithm` — 算法基类（所有算法模块继承此类）
- `MapCanvas3D(FigureCanvasQTAgg)` — 3D 地图画布
- `DispatchPage(QWidget)` — 调度页面

**界面布局：**
```
┌──────────────────────────────────────────────────┐
│  场景:[▼]  算法:[▼]  [开始调度]  状态:就绪  救援调度 │
├──────────────────────────────────────────────────┤
│                                                  │
│              3D 地图画布（滚轮缩放）                │
│              左键旋转  右键平移                     │
│                                                  │
├──────────────────────────────────────────────────┤
│  运行日志                                        │
└──────────────────────────────────────────────────┘
```

**调度流程：**
```
点击"开始调度"
  → 加载地图模块 (map_*.py)
  → 加载算法模块 (algo_*.py)
  → 读取物资/服务区/救援点数据
  → 调用 algo.solve(drones, materials, service_areas, rescue_points, map_obj)
  → 渲染地图 + 路径结果
  → 显示日志
```

**地图画布交互：**
- 左键拖拽：绕 Z 轴水平旋转（固定俯仰角 26°）
- 右键拖拽：平移画面
- 滚轮：缩放

**关键代码 — 算法调用：**
```python
result = self.current_algo.solve(
    drones,            # 无人机列表
    materials,         # 物资列表
    service_areas,     # 服务区列表
    rescue_points,     # 救援点列表
    self.current_map   # 地图对象（算法可通过它获取障碍物）
)
```

**算法返回值格式：**
```python
{
    "message": "调度完成",
    "total_time": 1200.0,       # 总时间（秒）
    "total_distance": 5000.0,   # 总距离（米）
```

### 7.2 `algo_hho.py` — 哈里斯鹰优化算法

**类名：** `Algorithm`（继承 `BaseAlgorithm`）

**原理：** 模拟哈里斯鹰捕猎行为，在解空间中搜索最优无人机-救援点分配方案。

**调用方式：**
```python
from algo_hho import Algorithm
algo = Algorithm()
result = algo.solve(drones, materials, service_areas, rescue_points, map_obj)
```

**参数：**
- 鹰群数量：min(50, max(20, n_drones × n_rp × 5))
- 迭代次数：100

---

### 7.3 `algo_rrt.py` — 改进 RRT* 算法

**类名：** `Algorithm`（继承 `BaseAlgorithm`）

**原理：** 快速探索随机树，在三维空间中随机采样构建路径树。
**改进：** 目标偏好采样 + 障碍物膨胀 + 路径平滑。

---

### 7.4 `algo_dqn.py` — DQN 强化学习算法

**类名：** `Algorithm`（继承 `BaseAlgorithm`）

**原理：** 将调度建模为马尔可夫决策过程，使用 Q-table 近似实现（无需深度学习框架）。

**状态空间：** 无人机位置、剩余物资、救援点状态
**动作空间：** 选择下一个目标救援点
**奖励函数：** 距离 + 优先级 + 时间窗

---

### 7.5 `algo_hybrid.py` — 混合启发式算法

**类名：** `Algorithm`（继承 `BaseAlgorithm`）

**原理：** 双阶段混合启发式协同调度。
- 第一阶段：任务分配（哪架无人机去哪个救援点）
- 第二阶段：路径规划（具体飞行路径）

## 10. 如何扩展

### 10.1 添加新地图

1. 创建 `map_xxx.py`，实现 `Map` 类继承 `BaseMap`
2. 实现以下方法：
   - `get_obstacles()` — 返回障碍物列表
   - `get_service_areas()` — 返回服务区列表
   - `get_rescue_points()` — 返回救援点列表
   - `get_bounds()` — 返回场景边界
   - `render_3d(ax)` — 渲染 3D 地图
3. 在 `config.py` 的 `DEFAULT_MAPS` 中添加配置
4. 在 `dispatch_page.py` 的 `_keep_supported_maps()` 中添加文件名

### 10.2 添加新算法

1. 创建 `algo_xxx.py`，实现 `Algorithm` 类继承 `BaseAlgorithm`
2. 实现以下方法：
   - `solve(drones, materials, service_areas, rescue_points, map_obj)` — 核心求解
   - `render_result(ax, result)` — 渲染结果（可选）
3. 在 `config.py` 的 `DEFAULT_ALGORITHMS` 中添加配置

### 10.3 算法中使用障碍物数据

```python
def solve(self, drones, materials, service_areas, rescue_points, map_obj):
    # 获取障碍物
    obstacles = map_obj.get_obstacles()
    
    # 获取场景边界
    (xmin, xmax), (ymin, ymax), (zmin, zmax) = map_obj.get_bounds()
    
    # 山区场景：获取地形高程
    if hasattr(map_obj, 'get_terrain_height'):
        import numpy as np
        x, y = np.array([100.0]), np.array([200.0])
        terrain_z = map_obj.get_terrain_height(x, y)[0]
    
    # 碰撞检测
    def is_collision(px, py, pz):
        for obs in obstacles:
            cx, cy, cz = obs["center"]
            w, h, d = obs["size"]
            if (cx-w/2 <= px <= cx+w/2 and 
                cy-h/2 <= py <= cy+h/2 and 
                pz <= d):
                return True
        # 山区还要检查地形
        if hasattr(map_obj, 'get_terrain_height'):
            tz = map_obj.get_terrain_height(
                np.array([px]), np.array([py])
            )[0]
            if pz < tz:
                return True
        return False
```
