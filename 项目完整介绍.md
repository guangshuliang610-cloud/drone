# 应急无人机调度系统 v2.1

> Emergency UAV Dispatch System — 面向应急响应的无人机物资配送协同调度与路径优化

---

## 项目简介

本系统是一个基于 **PyQt5 + Matplotlib 3D** 的桌面端应急无人机调度平台，面向灾害场景下的无人机物资配送问题。系统支持多场景三维地形建模、多算法路径规划对比、以及完整的救援资源（物资、服务区、救援点、无人机）管理功能。

用户可通过图形界面选择不同的灾害场景（山区/城市）和优化算法，系统自动计算无人机最优配送路径，并在 3D 地图上实时渲染飞行轨迹，实现从"灾情分析 → 资源管理 → 路径规划 → 任务调度"的完整闭环。

## 功能模块

### 1. 用户认证（login.py）

- 登录 / 注册双 Tab 界面，深色科技风格
- SHA-256 密码加密存储
- 记住密码功能（emembered.json）
- 用户数据存储于 users.json

### 2. 物资管理（material_page.py）

- 物资的增删改查
- 字段：名称、优先级（P0 紧急 ~ P3 低）、重量 (kg)、数量、所属服务区、配送救援点、备注
- **场景筛选**：支持按"山区避障场景"/"城市地震场景"分别管理
- 搜索 + 优先级筛选
- 默认预置 5 种物资（两场景各分配）

### 3. 服务区管理（area_page.py）

- 服务区的增删改查
- 字段：名称、三维坐标 (X, Y, Z)
- **场景筛选**：按场景分别管理，坐标范围自动适配
- 山区场景：X ∈ [-320, 320]，Y ∈ [-300, 300]
- 城市场景：X ∈ [-420, 420]，Y ∈ [-390, 390]

### 4. 救援点管理（rescue_page.py）

- 救援点的增删改查
- 字段：名称、三维坐标、配送优先级、备注
- **场景筛选**：同服务区
- **山区地形约束**：Z 坐标自动匹配地形高程（从 contour_data.json 读取等高线数据进行双线性插值），误差 ±5m
- 优先级颜色标识：P0 红色、P1 黄色、P2 绿色、P3 灰色

### 5. 无人机管理（drone_page.py）

- 设置无人机数量（1 ~ 100），自动增减
- 每架无人机属性：编号、名称、型号、最大载荷 (kg)、最大航程 (km)、最大速度 (km/h)、电量 (%)、状态（待命/工作中/维护中/离线）
- 表格内可直接编辑
- 批量状态切换
- 默认配置：4 架 DJI M30T（5kg 载荷、15km 航程、60km/h 速度）

### 6. 救援调度（dispatch_page.py）—— 核心模块

- **场景选择**：下拉框选择灾害场景（城市地震 / 山区避障）
- **算法选择**：下拉框选择路径规划算法（HHO / RRT* / DQN / 混合启发式）
- **开始调度**：一键运行，自动完成：
  1. 加载地图模块，渲染 3D 场景
  2. 加载算法模块，计算最优路径
  3. 在 3D 画布上叠加绘制飞行轨迹
- **运行日志**：实时输出调度过程信息
- **动态加载**：地图和算法通过 importlib 动态导入，新增场景/算法只需注册即可

### 7. 任务管理（task_page.py）

- 任务的创建、编辑、删除、状态管理
- 字段：任务名称、场景、算法、关联无人机、服务区、救援点、物资、优先级、状态
- 任务状态：待执行 → 执行中 → 暂停中 → 已完成 / 已取消
- 一键生成路由：调用所有算法为当前任务计算路径

## 灾害场景

### 城市地震场景（map_city_quake.py）

以**天津**为原型设计的北方城市震后场景：

- **地标建筑**：天津广播电视塔（天塔），高度 130m，位于场景中心
- **CBD 高层商务区**：4 栋现代写字楼（60 ~ 82m），东北方向分布
- **中层公共建筑**：10 栋（35 ~ 48m），沿主干道散布全城（天津站、南开大学、滨江道等）
- **传统居民区**：15 栋低层建筑（13 ~ 20m），四角网格排列，模拟天津老城街区
- **道路系统**：棋盘式主干道 + 次干道（模拟天津路网）
- **场景边界**：X ∈ [-420, 420]，Y ∈ [-390, 390]，Z ∈ [0, 160]

建筑配色：
| 建筑类型 | 颜色 | 说明 |
|----------|------|------|
| tower（天塔） | 金色 #E8C850 | 城市地标，醒目突出 |
| highrise（高层） | 钢蓝 #60B0D8 | 玻璃幕墙质感 |
| midrise（中层） | 暖灰 #C0A888 | 混凝土/砖混 |
| building（低层） | 浅暖灰 #D8C8B0 | 老城砖墙 |

### 山区避障场景（map_flood.py）

连续山脉地形 + 分层等高线的山地避障场景：

- **地形模型**：基于高斯函数叠加的数学地形（西部高地带、中央山脉、东北支脉、西南盆地）
- **等高线**：40 级分层渲染，绿色系地形色彩（低海拔深绿 → 高海拔浅绿）
- **8 座山峰障碍物**：分布在三个区域（西部 3 座、中部 3 座、东北 2 座）
- **地形高程函数**：get_terrain_height(x, y) 支持标量和数组输入
- **等高线导出**：get_contour_lines() 提取全部坐标，用于救援点 Z 坐标自动匹配
- **场景边界**：X ∈ [-320, 320]，Y ∈ [-300, 300]，Z ∈ [0, 220]

## 路径规划算法

系统内置 4 种算法，均继承 BaseAlgorithm 基类，接口统一：

`python
class Algorithm(BaseAlgorithm):
    def solve(self, drones, materials, service_areas, rescue_points, map_obj) -> dict
    def render_result(self, ax, result) -> None
`

### 1. 改进 HHO 算法（algo_hho.py）

- **全称**：哈里斯鹰优化算法（Harris Hawks Optimization）
- **原理**：模拟哈里斯鹰的协同捕猎行为
  - 探索阶段：随机搜索全局空间
  - 开发阶段：围捕猎物（全局最优解）
- **改进**：自适应逃逸能量 + Lévy 飞行增强全局搜索能力
- **适用**：全局路径寻优，适合大规模配送网络

### 2. 改进 RRT* 算法（algo_rrt.py）

- **全称**：快速探索随机树星（Rapidly-exploring Random Tree Star）
- **原理**：在三维空间中随机采样，逐步构建路径树
- **改进**：目标偏好采样 + 障碍物膨胀 + 路径平滑
- **适用**：三维动态避障，适合复杂地形环境

### 3. DQN 强化学习（algo_dqn.py）

- **全称**：Deep Q-Network
- **原理**：将调度问题建模为马尔可夫决策过程
  - 状态：当前无人机位置、剩余物资、救援点状态
  - 动作：选择下一个目标救援点
  - 奖励：综合距离、优先级、时间窗
- **实现**：Q-table 近似（无需深度学习框架）
- **适用**：智能决策与路径规划

### 4. 混合启发式算法（algo_hybrid.py）

- **第一阶段**：遗传算法（GA）进行全局任务分配（哪架无人机 → 哪个救援点）
- **第二阶段**：蚁群算法（ACO）进行局部路径优化（经过哪些航点）
- **适用**：双阶段协同调度，结合全局搜索和局部优化优势

## 已知限制

- PyQt5 的 sipPyTypeDict() 警告为版本兼容性提示，不影响功能
- Matplotlib 3D 渲染在大量障碍物时可能有性能瓶颈
- 当前算法实现基于 Q-table 近似，非深度神经网络版本
- 数据存储采用 JSON 文件，不适合大规模并发场景
